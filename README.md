# 🚀 C++ Learning Journey: Dai Fondamenti alla Programmazione a Oggetti (OOP)

Benvenuto nel repository principale dedicato al mio percorso di studio, sperimentazione e approfondimento continuo con il linguaggio **C++**. 

Questo spazio raccoglie tutti i progetti, gli algoritmi e le strutture software sviluppati per padroneggiare la logica del linguaggio, l'ottimizzazione dei dati e i paradigmi avanzati dell'Ingegneria del Software. Il codice è organizzato in moduli logici che tracciano una crescita costante dalle basi fino all'architettura software avanzata.

---

## 🗺️ Mappa del Percorso (Struttura del Repository)

Il repository è in continuo aggiornamento ed è suddiviso in tre macro-moduli principali:

| Modulo / Cartella | Focus Didattico | Contenuti Principali |
| :--- | :--- | :--- |
| **📋 1. Fondamenti** | Logica di programmazione base | Variabili, input/output, strutture di controllo (`if`/`else`, cicli) e scomposizione in funzioni. |
| **📊 2. Matrici e Strutture Dati** | Gestione della memoria e liste | Array statici, matrici bidimensionali e utilizzo dei vettori dinamici della Standard Template Library (`std::vector`). |
| **🧠 3. Oggetti (OOP)** | Architettura e modellazione software | Paradigmi della programmazione a oggetti: Incapsulamento, Ereditarietà, Astrazione e Polimorfismo Dinamico. |

---

## 📂 Panoramica dei Moduli

### 🔹 1. Fondamenti
Il punto di partenza del percorso, incentrato sulla padronanza del flusso dei dati, sulla validazione dell'input e sul riutilizzo del codice (approccio DRY - *Don't Repeat Yourself*) mediante funzioni isolate con parametri passati per valore o riferimento.

### 🔹 2. Matrici e Strutture Dati
Sezione dedicata all'organizzazione di collezioni di dati complesse. Esplora la manipolazione di griglie bidimensionali e l'uso efficiente di contenitori dinamici per la gestione di liste di elementi variabili a runtime.

### 🔹 3. Oggetti (OOP)
Il modulo più avanzato del framework, mirato alla modellazione della realtà tramite classi. Copre la protezione dei dati sensibili mediante l'uso di specificatori d'accesso (`private`/`protected`), le gerarchie di ereditarietà, le interfacce astratte e la gestione sicura della memoria dinamica tramite puntatori e distruttori virtuali.
